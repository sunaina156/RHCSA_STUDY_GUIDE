## 🎯 Objective
Understand how to create, copy, move, rename, and delete files and directories safely in Linux while following production best practices.

---

- Everything in Linux is treated as a file, including directories and devices
- Directories are special files that store references to other files and directories
- `cp` creates a copy while keeping the original file intact
- `mv` is used for both moving and renaming files/directories
- `touch` creates empty files or updates timestamps
- `mkdir` creates directories and `mkdir -p` creates nested directory structures
- `rm` permanently deletes files without a recycle bin
- `rmdir` removes only empty directories
- Recursive operations (`-r`) apply to all files and subdirectories
- Hidden files begin with a dot (`.`) and are displayed using `ls -a`
- Production administrators should always verify files before deletion
- Absolute paths are safer than relative paths in scripts and automation

---

# 💻 Commands Used

```bash
touch file.txt              # Creates an empty file or updates timestamp
mkdir testdir              # Creates a directory
mkdir -p app/logs/archive  # Creates nested directories
cp file1 file2             # Copies a file
cp -r source backup        # Copies a directory recursively
cp -i file1 file2          # Prompts before overwriting
cp -v file1 file2          # Shows copy operation details
mv file1 file2             # Renames file
mv file1 /tmp             # Moves file
rm file.txt               # Deletes file permanently
rm -r directory           # Deletes directory recursively
rm -rf directory          # Force deletes directory and contents
rmdir directory           # Removes empty directory
ls -a                     # Shows hidden files
ls -l                     # Displays detailed file information
```

# Troubleshooting Notes

### 1. cp: omitting directory

Cause:
Directory copied without recursive option.

Fix:

```bash
cp -r source_dir backup_dir
```

---

### 2. Directory not empty

Cause:
Attempting to remove non-empty directory using `rmdir`.

Fix:

```bash
rm -r directory
```

---

### 3. File accidentally overwritten

Cause:
Using `cp` without verification.

Fix:

```bash
cp -i source destination
```

---

### 4. Important files deleted

Cause:
Incorrect use of:

```bash
rm -rf
```

Prevention:

```bash
pwd
ls
```

Verify location before deletion.

---

### 5. Files copied to wrong location

Cause:
Incorrect path usage.

Fix:
Use absolute paths in scripts and automation.

Example:

```bash
/opt/application/config
```

instead of:

```bash
config
```

# Real-World Production Scenarios

### 1. Application Deployment

Deploying a new application version.

Best Practice:

- Backup existing files
- Verify destination directory
- Copy deployment artifacts
- Validate permissions

Example:

```bash
cp app.jar /opt/application/
```

---

### 2. Configuration Backup

Before modifying configuration files:

```bash
cp nginx.conf nginx.conf.bak
```

Always maintain rollback capability.

---

### 3. Log Cleanup

Large log directories consume disk space.

Verify before deletion:

```bash
du -sh *
ls
```

Avoid:

```bash
rm -rf *
```

without verification.

---

### 4. CI/CD File Distribution

Deployment pipelines copy artifacts to target servers.

Example:

```bash
cp release.war /opt/tomcat/webapps/
```

Always verify destination path and file ownership.

---

### 5. Archive and Retention Operations

Move old logs and backups into archive locations.

Example:

```bash
mv *.log archive/
```

Used frequently in automation and maintenance scripts.
