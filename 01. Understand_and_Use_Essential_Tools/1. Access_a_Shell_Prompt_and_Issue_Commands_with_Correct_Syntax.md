uname -r      # Shows Linux kernel version
uname -a      # Shows full system information
echo $SHELL   # Displays current shell
echo $PATH    # Shows directories where commands are searched
env           # Shows all environment variables
which ls      # Shows full path of a command
type ls       # Shows whether command is alias, builtin, or binary
history       # Shows previously executed commands
clear         # Clears terminal screen


-------------------------------------------------------------------------------------------------------------------------------------------------
Issues Faced & Fixes

Issue 1: Command not found
Fix:
Checked if command exists using which
Verified PATH using echo $PATH
Used full path when required (e.g. /usr/bin/ls)

Issue 2: Confusion between shell and terminal
Fix:
Terminal = interface
Shell = command processor

Issue 3: Command works manually but not in automation
Fix:
Learned cron/automation uses limited PATH
Used absolute path in scripts


-------------------------------------------------------------------------------------------------------------------------------
Real-World Scenarios

Scenario 1: Cron Job Fails
A cron job fails even though command works in terminal.
Reason:
Cron runs with minimal environment and limited PATH.

Fix:
Use full path:
/usr/bin/aws s3 cp file.txt s3://bucket


Scenario 2: Command Not Found in Production
Application fails with "command not found".
Reason:
Binary not in PATH or not installed.

Fix:
Check using which
Fix PATH or install package


Scenario 3: Script Works for Root but Not User
Reason:
Different PATH and permissions.

Fix:
Check user environment
Use absolute paths
Verify permissions


Scenario 4: Application Binary Not Running
Binary exists but command not found.
Example:
/opt/tools/myapp

User types:
myapp

Fix:
Add /opt/tools to PATH OR
Create symlink in /usr/bin
