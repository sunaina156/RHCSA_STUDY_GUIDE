## 🎯 Objective
Learn how to use Linux system documentation effectively to understand commands, troubleshoot issues, and work independently on production servers without relying on internet access.

---

- Linux provides built-in documentation for commands and installed software
- `man` pages are the primary source of command documentation
- `info` pages provide more detailed and structured documentation than man pages
- `/usr/share/doc` contains package-specific documentation files
- Most Linux commands have documentation available locally
- Man pages contain sections such as NAME, SYNOPSIS, DESCRIPTION, OPTIONS, and EXAMPLES
- `whatis` provides a short description of a command
- `apropos` helps find commands when the exact command name is unknown
- Documentation is a critical troubleshooting tool in production environments
- Production servers may not have internet access, making local documentation essential
- Experienced Linux administrators rely on documentation rather than memorizing every command

---

# 💻 2. Commands Used

```bash
man ls              # Opens manual page for ls command
man -k copy         # Searches man pages using a keyword
info ls             # Opens detailed info documentation
whatis ls           # Displays one-line command description
apropos copy        # Finds commands related to a keyword
ls /usr/share/doc   # Lists installed package documentation
```

Troubleshooting Notes (Merged & Realistic)

1. Command usage is unknown
Fix:
```bash
man command_name
```

2. Command name is forgotten
Fix:
```bash
apropos keyword
```

3. Need a quick description of a command
Fix:
```bash
whatis command_name
```

4. No internet access on production server
Fix:
Use:
```bash
man command_name
info command_name
ls /usr/share/doc
```

5. Command behavior is confusing
Fix:
Read command documentation before making assumptions:
```bash
man command_name
```
