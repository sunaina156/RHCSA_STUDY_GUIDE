## 🎯 Objective
Understand Linux multi-user system, user switching, privilege management, and real-world production troubleshooting.

---

- Linux is a multi-user operating system where multiple users can work simultaneously
- Each user has separate permissions, environment variables, and processes
- Root user has full administrative control over the system
- Normal users have restricted access for security
- `su` switches user but keeps current environment
- `su -` switches user with full login environment (recommended)
- `sudo` runs commands with elevated privileges without switching user
- `sudo -i` opens a full root shell session
- `/etc/sudoers` defines sudo permissions for users
- Multi-user systems allow monitoring of active sessions
- Environment differences often cause command failures
- Production systems avoid direct root login for security reasons

# 💻 2. Commands Used

```bash id="cmd-section"
whoami     # Shows current logged-in user
id         # Displays UID, GID, and group information
who        # Lists all users currently logged into system
w          # Shows active users and their running processes
su user    # Switches to another user (keeps environment)
su - user  # Switches user with full login environment
sudo cmd   # Runs a command as root (if permitted)
sudo -i    # Opens root shell session
exit       # Exits current user session or shell

Troubleshooting Notes (Merged & Realistic)

1. sudo not working (permission denied)
User not added to sudo/wheel group
Fix:
usermod -aG wheel username

2. Environment not updating after switching user
Happens when using su instead of su -
Fix: always use full login shell
su - username

3. Command works for root but not normal user
Caused by PATH or permission mismatch
Fix:
echo $PATH

4. Permission denied error
Caused by file ownership or incorrect permissions
Fix:
ls -l

Real-World Production Scenarios (Merged)

1. Scenario 1: Secure server access
Root login disabled in production for security
Admin uses:
sudo -i

2. Scenario 2: Multi-user monitoring
Multiple engineers logged into same server
Used commands:
who
w

3.  Scenario 3: Application deployment failure
App works for root but fails for normal user
Cause: permission issue
Fix:
chown appuser:appgroup file
chmod 755 file

4. Scenario 4: Wrong environment issue
Using su user causes inconsistent environment
Fix:
su - user
